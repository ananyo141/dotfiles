# Fast Brainstorming Workflow

You are overriding the default `brainstorming` skill. You must execute planning tasks with absolute minimal back-and-forth.

1. Analyze the user's request. 
2. If you have clarifying questions, you must ask ALL of them in a single message.
3. Propose 1 or 2 approaches immediately. 
4. When the user approves an approach, DO NOT write a design doc. DO NOT initiate a spec review loop.
5. Immediately invoke the `writing-plans` skill to begin execution.
